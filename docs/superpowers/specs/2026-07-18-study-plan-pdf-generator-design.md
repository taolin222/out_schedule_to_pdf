# 公考学习计划 PDF 生成器 — 设计文档

**日期：** 2026-07-18
**项目：** out_schedule_to_pdf
**平台：** iOS（Flutter）

---

## 1. 概述

一个纯离线的 iOS App，用户填写考试日期、计划日期、言语和判断推理的子项目内容，基于 Word 模板样式生成一张可打印的学习计划 PDF，支持预览和分享。

---

## 2. 目标用户

准备公务员考试（国考/省考）的考生，需要打印每日学习计划表来手写记录学习进度。

---

## 3. 技术方案

- **框架：** Flutter（已有项目）
- **PDF 生成：** `pdf` 包（Dart 纯代码生成 PDF）
- **PDF 预览与分享：** `printing` 包
- **中文字体：** 打包 Noto Sans SC（开源，可商用）
- **数据持久化：** 简单本地持久化，保存上次输入的言语和判断推理子项目内容，下次打开 App 自动恢复

---

## 4. 页面流程

```
首页 (数据录入) → 预览页 (PDF预览) → 分享/打印
```

### 4.1 首页 — 数据表单

包含以下输入项：

| 字段 | 类型 | 说明 |
|---|---|---|
| 考试日期 | DateTime | 日期选择器，用于计算距考试天数 |
| 计划日期 | DateTime | 日期选择器，自动推导年月日、星期 |
| 言语子项目 | String (多行) | 每行一个子项目，默认值：逻辑填空、语句衔接、片段阅读 |
| 判断推理子项目 | String (多行) | 每行一个子项目，默认值：类比推理、图推、定义判断、逻辑判断 |

所有其他字段（完成情况、做题时间、复盘时间、总用时、数量、资料分析、政治理论、常识、申论、总结心得等）**保留在 PDF 模板上但留空**，打印后手写。

### 4.2 预览页

- 使用 `printing` 包的 `PdfPreview` Widget 渲染 PDF
- 支持缩放、滚动
- 底部操作栏：保存、分享（系统分享菜单 → AirDrop/微信/打印等）

---

## 5. 数据模型

```dart
class StudyPlan {
  final DateTime examDate;
  final DateTime planDate;
  final String verbalItems;    // 换行分隔的子项目
  final String reasoningItems; // 换行分隔的子项目
}
```

**自动计算字段（生成 PDF 时计算，不持久化）：**

| 字段 | 计算方式 |
|---|---|
| 月 | planDate.month |
| 日 | planDate.day |
| 星期 | planDate.weekday → 中文（周一~周日） |
| 距考试天数 | examDate.difference(planDate).inDays |

---

> 申论下的小题第二列跨三行合并居中；所有单元格内容水平垂直居中。

### 6.2 动态行逻辑

- **言语：** 根据 `verbalItems` 的换行拆分为子项行，每行一个子项名称，第一列"言语"跨行合并居中
- **判断推理：** 同理，根据 `reasoningItems` 拆分，第一列"判断推理"跨行合并居中
- 其他模块（数量、资料分析、政治理论、常识、申论）：**固定行**，名称保留但内容单元格全部留空；分类名称所在的列跨行合并居中

### 6.3 字体

- 打包 Noto Sans SC（提供 Regular 和 Bold 两种字重）
- 标题使用 Bold，正文使用 Regular

## 8. 边界情况与错误处理

| 场景 | 处理方式 |
|---|---|
| 计划日期 > 考试日期 | 计划日期选择器限制只能选 ≤ 考试日期的日期 |
| 计划日期 == 考试日期 | 距考试天数为 0，显示"今天考试" |
| 言语/判断推理子项目为空 | 对应模块只显示标题行，不插入子项行 |
| 子项目行数过多 | 自动调整行高，内容超出分页（暂不分页，保持一页） |
| 未选择日期 | 禁用生成按钮，提示选择日期 |

## 9. 非功能需求

- **离线运行：** 无需网络连接，所有计算在本地完成
- **启动速度：** App 应在 2 秒内可操作
- **包体积：** 中文字体（Noto Sans SC Regular + Bold）约 10-15MB，在可接受范围内
- **隐私：** 不收集任何用户数据，不上传任何内容
